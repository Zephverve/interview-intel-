## 五、按题型

### 项目深挖

- P0-1 架构白板  
- 为什么单独封装 Python 服务 / 抽象接口有哪些  
- 项目最复杂的一件事是什么  
- 共享屏幕讲架构  

### Agent 原理

- P0-2 ReAct 循环与实现  
- P0-6 Workflow vs Agent 选型  
- ReAct vs CoT vs ToT vs Plan-and-Execute  
- 规划器路径震荡怎么避免  
- Agent 死循环 / max_turns / 重复检测  

### 协议与工具

- P0-4 MCP / FC / Skill / A2A  
- MCP 协议内部结构（Host-Client-Server）  
- 工具调用失败 / 参数校验 / 幻觉 tool name  
- P1-6 工具过多与 Skill Routing  

### Memory & RAG

- P0-7 长短期记忆  
- Memory 和 RAG 区别  
- 多用户并发 memory 隔离  
- P1-2 混合检索、重排、query 改写  
- 知识库过期 / 错误内容治理  
- embedding 检索 vs 规则知识库（.catpaw/rules vs RAG memory）  

### 系统设计

- P0-3 Agent 平台 0→1  
- P0-8 企业智能客服 Agent  
- 百万用户 / 高 QPS 哪里会崩  
- 无状态 worker + Redis session  
- 超时、熔断、降级、幂等  
- 流式输出（SSE）与多轮 LLM 交互  

### 工程 & 评测

- P0-5 失败场景与重试安全  
- P1-4 Harness（沙箱、checkpoint、trace）  
- P1-5 RAGAS / DeepEval / golden set  
- Evaluation 模块为何独立  
- Token 成本优化（Context Caching、小模型路由）  

### Multi-Agent

- P1-3 主从通信、决策冲突  
- 为什么 Multi-Agent 不封装成 Skill  
- Boss-Worker 模式、任务分解粒度  

### 后端 & 计算机基础（Agent 岗仍考）

- Redis 用途、Lua 脚本、分布式锁、看门狗  
- MySQL 索引设计（B+ 树）  
- 输入 URL 到页面渲染发生了什么  
- HTTP 响应是一次性还是流式  
- 排序算法（快排、归并等）  

### 行为 / HR

- 遇到的最大挑战  
- 为什么不用传统 Workflow  
- 对 MiniMax / DeepSeek / 字节 Agent 产品的理解  
- 反问：团队场景、评测体系、实习 ownership
