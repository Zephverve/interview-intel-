# 简历交叉分析

## 提取字段

```yaml
basic: {姓名, 学历, 年限}
skills: [Python, LangGraph, ...]
projects:
  - name, tech, claims（量化指标）, duration
job_intent: 意向岗位
```

只提取简历明确内容，不臆造。

## 三维匹配（输出格式）

### ✅ 强匹配 · 必被深挖

面经高频 ∩ 简历项目/技能（重叠 ≥2 项）

输出：预测追问 + 必备 data point（指标、架构决策、踩坑）

### ⚠️ 薄弱项 · 补洞

面经 P0/P1 有，简历无对应

输出：补读什么 / 练什么 demo / 或「为什么没用 X」的话术

### 💡 优势项 · 主动引导

简历亮点但面经出现少

输出：自我介绍或「还有什么补充」时主动抛出的故事

## 表格模板

| 简历项 | 面经关联 | 预测追问 | 必备 data point |
|--------|----------|----------|-----------------|
| GenericAgent | Agent Memory | L0-L4 怎么分层 | token 对比 |

## 风险扫描

| 表述 | 风险 | 应对 |
|------|------|------|
| 参与/协助为主 | 贡献度 | 明确个人模块 |
| 只有框架名 | 调包 | 准备 1 个自改点 |
| 无 baseline 指标 | 不可信 | 补 before/after |
| 项目 >4 个 | 每个都浅 | 主推 2 个 |

## 无简历

输出通用 **STAR + 追问链** 模板（Agent 项目链、RAG 项目链各一套）。

---

## 分轮次个性化预测（有简历 · 必出）

从简历提取 2 个主推项目（Agent 岗通常：Agent 框架 + RAG/问答系统），按轮次映射面经 P0/P1。

### 一面（5–8 题）

- 自我介绍（引导到主推项目）
- 主推项目 1 概述 + 技术栈验证（与简历 bullet 一致）
- 面经 P0 中与该项目交集的 2–3 题（如 ReAct、Memory、RAG）
- 基础八股 1–2 题（Transformer / Redis / HTTP，视面经）
- 算法 1 题（Easy/Medium）

每题标注：**简历哪一行会被问到**。

### 二面（5–8 题）

- 主推项目 1 深挖（架构图、指标、失败 case）
- 主推项目 2 或次要项目对比
- 系统设计（与简历场景结合，如「设计你的科研问答 Agent 服务」）
- 面经 P0 系统设计 / Memory / 评测

### 三面（3–5 题）

- 架构取舍（为什么 LangGraph vs 自研 ReAct）
- 简历 bold claim 验证（如「分层记忆优于向量检索」）
- 行业/团队/职业规划

### HR（3–4 题）

- 动机、base、实习时间、稳定性
- 一般不深挖技术，除非简历空档
