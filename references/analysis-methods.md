# 聚合与分析方法

## 去重与标准化

1. 语义相似题合并（「手写 LRU」=「实现 LRU 缓存」）
2. 统一为规范表述
3. 记录：出现 **篇数** + **公司列表**

## P0 / P1 / P2 判定

结合面经频次与岗位 must_cover（role-keywords.md）：

| 级别 | 规则 | 数量目标 |
|------|------|----------|
| **P0** | ≥3 篇 或 ≥2 家公司 或 must_cover 核心 | 8–12 |
| **P1** | 2 篇 或 简历强 match | 10–15 |
| **P2** | 1 篇 / 新趋势 / 加分 | 5–10 |

**low 可信度面经**中的题不参与 P0，仅 P2 参考。

标签：`[面经原文]` · `[推断]` · `[通用共识]`（P0 至少 60% 有面经支撑）

## 分类型

| 类型 | 典型内容 |
|------|----------|
| 算法 | LeetCode、手写数据结构 |
| 八股/基础 | 语言、框架、LLM 原理 |
| 系统设计 | 高并发、Agent 服务、IM |
| 项目深挖 | 简历项目、技术选型、指标 |
| 行为 | 挑战、协作、冲突 |
| HR | 动机、薪资、稳定性 |

## 分轮次（面无标注时推断）

| 轮次 | 常见内容 |
|------|----------|
| 一面 | 算法 + 基础 + 简历初筛 |
| 二面 | 系统设计 + 项目深挖 |
| 三面 | 架构/综合/总监 |
| HR | 动机 + 文化 + 薪资 |

## 趋势分析

单独列出 **近 3 个月** 新热点，例如：
- MCP / Skill / Harness Engineering
- Multi-Agent / A2A
- 规划器路径震荡 / Skill Routing
- 从「会不会调 API」→「生产级 Agent 系统设计」

## 公司范围与报告

- **不限**：高频表注明各题来自大厂/中小/外企样本
- **单一范围**：过滤样本，标题写清楚范围
- **指定公司**：公司差异节只写该公司特点
